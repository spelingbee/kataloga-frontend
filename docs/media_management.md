# Управление изображениями товаров

В нашем приложении изображения товаров обрабатываются динамически через API. Вот как это устроено и как их изменить.

## 1. Откуда берутся фото?
Каждый товар в меню имеет свойство `imageUrl`. 
- **Источник**: Эти ссылки хранятся в базе данных и приходят с бэкенда при запросе `/api/menu`.
- **Заглушки**: Если у товара нет фото (поле `imageUrl` пустое), приложение автоматически показывает заглушку из файла `public/images/placeholder-dish.svg`.

## 2. Как заменить фото?

### Вариант А: Через Админ-панель (Рекомендуется)
Если у вас есть доступ к панели управления тенантом:
1. Перейдите в раздел **Меню** или **Товары**.
2. Выберите нужный товар.
3. Загрузите новое фото или вставьте ссылку на него.
4. Сохраните изменения. Фронтенд обновит фото при следующей загрузке страницы.

### Вариант Б: Через JSON-данные (Для разработчиков)
Если вы правите данные вручную (через файлы сидов или напрямую в БД):
1. Найдите объект товара.
2. Измените значение поля `imageUrl` на прямую ссылку на изображение (например, `https://your-storage.com/photo.jpg`).

### Вариант В: Локальные изображения
Если вы хотите использовать локальные файлы для тестов:
1. Положите файл в папку `apps/frontend/public/images/products/my-burger.jpg`.
2. В данных товара укажите путь: `/images/products/my-burger.jpg`.

## 3. Технические детали
- **Компонент**: За отображение отвечает `MenuItemImage.vue`.
- **Обработка ошибок**: Если ссылка на фото битая, компонент автоматически переключится на иконку-заглушку, чтобы интерфейс не ломался.
- **Оптимизация**: Изображения загружаются «лениво» (`loading="lazy"`), что экономит трафик пользователей на мобильных устройствах.
